# RestDB.io Database Setup for FairShare

This document details the database structure required for FairShare when using RestDB.io as the backend database.

## Overview

FairShare uses four main collections:
1. **groups** - Expense sharing groups
2. **members** - Group members with access tokens
3. **bills** - Expense records
4. **splits** - Individual payment shares for each bill

## Collection Schemas

### 1. groups Collection

**Collection Name:** `groups`

| Field Name | Type | Required | Description |
|------------|------|----------|-------------|
| `_id` | string | Yes (auto) | Unique identifier (auto-generated by RestDB) |
| `name` | string | Yes | Name of the group |
| `organizerId` | string | No | Reference to the member who is the organizer |
| `config` | JSON | No | Group configuration settings object |

**Note:** The `config` field stores a JSON object. You only need to create the `config` field itself - do NOT create separate fields for `config.requireDates` or `config.requireMemberSelection`. These are properties within the JSON object that will be stored automatically.

**Example Document:**
```json
{
  "_id": "abc123",
  "name": "Holiday Trip 2026",
  "organizerId": "mem456",
  "config": {
    "requireDates": true,
    "requireMemberSelection": true
  }
}
```

### 2. members Collection

**Collection Name:** `members`

| Field Name | Type | Required | Description |
|------------|------|----------|-------------|
| `_id` | string | Yes (auto) | Unique identifier (auto-generated by RestDB) |
| `groupId` | string | Yes | Reference to the group this member belongs to |
| `name` | string | Yes | Member's display name |
| `email` | string | No | Member's email address (optional) |
| `secureToken` | string | Yes | Unique access token for authentication |
| `joinDate` | string (ISO 8601) | Yes | Date the member joined the group |
| `leaveDate` | string (ISO 8601) | No | Date the member left the group (null if still active) |
| `isTokenActive` | boolean | Yes | Whether the member's access token is active |

**Example Document:**
```json
{
  "_id": "mem456",
  "groupId": "abc123",
  "name": "Alice Johnson",
  "email": "alice@example.com",
  "secureToken": "5a1b2c3d4e5f",
  "joinDate": "2026-05-01T00:00:00.000Z",
  "leaveDate": null,
  "isTokenActive": true
}
```

### 3. bills Collection

**Collection Name:** `bills`

| Field Name | Type | Required | Description |
|------------|------|----------|-------------|
| `_id` | string | Yes (auto) | Unique identifier (auto-generated by RestDB) |
| `groupId` | string | Yes | Reference to the group |
| `payerId` | string | Yes | Reference to the member who paid the bill |
| `purpose` | string | Yes | Description of the expense |
| `totalAmount` | number | Yes | Total amount in pence (e.g., 5000 = £50.00) |
| `applicablePeriodStart` | string (ISO 8601) | No | Start date of the billing period |
| `applicablePeriodEnd` | string (ISO 8601) | No | End date of the billing period |
| `dateLogged` | string (ISO 8601) | Yes | Date the bill was logged |
| `dateDue` | string (ISO 8601) | Yes | Payment due date |

**Example Document:**
```json
{
  "_id": "bill789",
  "groupId": "abc123",
  "payerId": "mem456",
  "purpose": "Electricity Bill - May",
  "totalAmount": 12500,
  "applicablePeriodStart": "2026-05-01T00:00:00.000Z",
  "applicablePeriodEnd": "2026-05-31T23:59:59.999Z",
  "dateLogged": "2026-05-20T10:30:00.000Z",
  "dateDue": "2026-06-10T00:00:00.000Z"
}
```

### 4. splits Collection

**Collection Name:** `splits`

| Field Name | Type | Required | Description |
|------------|------|----------|-------------|
| `_id` | string | Yes (auto) | Unique identifier (auto-generated by RestDB) |
| `billId` | string | Yes | Reference to the bill |
| `memberId` | string | Yes | Reference to the member who owes this split |
| `amountOwed` | number | Yes | Amount owed in pence |
| `isPaid` | boolean | Yes | Whether this split has been paid |

**Example Document:**
```json
{
  "_id": "split101",
  "billId": "bill789",
  "memberId": "mem457",
  "amountOwed": 4167,
  "isPaid": false
}
```

## Setup Instructions

### Step 1: Create RestDB.io Account

1. Go to https://restdb.io
2. Sign up for a free account (or use an existing account)
3. Create a new database (e.g., "fairshare-prod")

### Step 2: Create Collections

For each of the four collections listed above:

1. Click "Collections" in the left sidebar
2. Click "+ New Collection"
3. Enter the collection name (exactly as specified: `groups`, `members`, `bills`, `splits`)
4. Add fields according to the schema tables above

**Fields to Create for Each Collection:**

**groups:** `name` (Text), `organizerId` (Text), `config` (JSON)  
**members:** `groupId` (Text), `name` (Text), `email` (Text), `secureToken` (Text), `joinDate` (Text), `leaveDate` (Text), `isTokenActive` (Boolean)  
**bills:** `groupId` (Text), `payerId` (Text), `purpose` (Text), `totalAmount` (Number), `applicablePeriodStart` (Text), `applicablePeriodEnd` (Text), `dateLogged` (Text), `dateDue` (Text)  
**splits:** `billId` (Text), `memberId` (Text), `amountOwed` (Number), `isPaid` (Boolean)

**Important Field Settings:**

- For `_id` fields: These are auto-generated by RestDB - do not create them manually
- For boolean fields: Set type to "Boolean"
- For number fields: Set type to "Number"
- For string fields: Set type to "Text"
- For JSON fields (like `config`): Set type to "JSON" - RestDB will store any valid JSON object you send
- For date fields: Set type to "Text" (we store ISO 8601 strings)

**Note on JSON Fields:** When you create a JSON field type, you only create the field itself (e.g., `config`). You do NOT define the internal structure or nested properties - RestDB accepts and stores any valid JSON object in that field.

### Step 3: Configure API Access

1. Go to "Settings" → "API" in RestDB dashboard
2. Note your Database URL (e.g., `https://fairshare-xxxx.restdb.io/rest`)
3. Create an API key with Read/Write permissions
4. Copy your API key

### Step 4: Configure Local Environment

1. Copy `.env.example` to `.env` in your project root:
   ```bash
   cp .env.example .env
   ```

2. Edit `.env` and add your RestDB credentials:
   ```env
   RESTDB_URL=https://fairshare-xxxx.restdb.io/rest
   RESTDB_KEY=your_actual_api_key_here
   ```

3. Save the file

### Step 5: Test the Connection

1. Start the local server:
   ```bash
   npm start
   ```

2. Open http://localhost:3000 in your browser
3. Create a test group
4. If it works, data should be stored in RestDB instead of `database.json`

## API Endpoints

The app interacts with RestDB through these endpoints:

- **GET** `/rest/groups?q={"_id":"abc123"}` - Get group by ID
- **POST** `/rest/groups` - Create new group
- **PUT** `/rest/groups/{id}` - Update group
- **GET** `/rest/members?q={"groupId":"abc123"}` - Get members by group
- **POST** `/rest/members` - Create new member
- **PUT** `/rest/members/{id}` - Update member
- **GET** `/rest/bills?q={"groupId":"abc123"}` - Get bills by group
- **POST** `/rest/bills` - Create new bill
- **GET** `/rest/splits?q={"billId":"bill789"}` - Get splits by bill
- **POST** `/rest/splits` - Create new split
- **PUT** `/rest/splits/{id}` - Update split

All requests must include the `x-apikey` header with your API key.

## Data Migration

To migrate existing data from `database.json` to RestDB:

1. Ensure `database.json` contains your local data
2. Configure RestDB credentials in `.env`
3. Use the RestDB API or web interface to manually import data
4. Verify all data migrated correctly
5. Remove or backup `database.json`

## Security Considerations

- **Never commit `.env` file** - It contains secret credentials
- **API Key Permissions** - Use separate API keys for development and production
- **CORS Settings** - Configure allowed origins in RestDB settings
- **Rate Limiting** - RestDB free tier has API rate limits - monitor usage

## Troubleshooting

### Connection Issues

If you get connection errors:
1. Verify RESTDB_URL is correct (must include `/rest` at the end)
2. Check that RESTDB_KEY is valid
3. Ensure your IP is not blocked by RestDB
4. Check RestDB dashboard for API usage limits

### Data Not Appearing

If data isn't appearing:
1. Check browser console for errors
2. Verify collections exist with correct names
3. Check RestDB "Logs" tab for API errors
4. Ensure API key has read/write permissions

### Falling Back to Local Storage

The app automatically falls back to `database.json` if:
- No RestDB credentials are configured
- RestDB connection fails
- RESTDB_URL or RESTDB_KEY environment variables are missing

## Email Service Configuration (Optional)

FairShare supports sending welcome emails via Resend.com:

1. Sign up at https://resend.com
2. Get your API key
3. Add to `.env`:
   ```env
   RESEND_API_KEY=re_your_key_here
   EMAIL_FROM=FairShare <noreply@yourdomain.com>
   ```

Without Resend configured, welcome emails will be logged to console only.
